## Documentation [20 marks]

| Grade | What's needed |
|---|---|

| Good (55%+) | Personas cover a wide range of users; user stories capture full system; acceptance criteria linked to tests |
| **Excellent (70%+)** | Personas clearly impact all parts of the system; evidence that priorities/stories evolved with justification; testing strategy built around personas |

### UX & UI Design [7]

| Grade | What's needed |
|---|---|
| Pass (40%) | Annotated wireframes with colour/icon rationale; at least one UX test with feedback acted on |
| Good (55%+) | Incremental design changes with clear reasoning; multiple UX tests |
| **Excellent (70%+)** | Full justification of every significant design change; UX testing integrated throughout project; clear link from testing to refactoring |

### Meeting Notes [2]

- **Pass:** Concise notes with discussion points and actions; retrospectives recorded; all team members contribute to minutes
- **Good:** Action tracking with outcomes monitored; outcomes of retrospectives feed into future retrospectives

- **Pass:** Branches used sensibly; no broken code on main
- **Good:** Consistent strategy; issues tracked and assigned to users

## Key Themes for Excellence

1. **Everything traces back to personas** — your design choices, testing strategy, and demo should all connect to your defined users
2. **Show evolution, not just a final snapshot** — document that things changed over time with justification
3. **CI/CD via GitHub Actions** — explicitly called out for top marks in both Git usage and Testing
4. **Zero accessibility errors** — run WCAG checks on the final system
5. **Security** — input validation and security testing are specifically mentioned for top marks in Functionality
6. **Acknowledge AI use** — add comments to any AI-assisted code stating which lines and which model was used
